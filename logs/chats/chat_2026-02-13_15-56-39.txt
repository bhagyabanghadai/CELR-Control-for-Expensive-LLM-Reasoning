Merge sort and quicksort are two popular sorting algorithms used to arrange data in ascending or descending order. But which one be faster?

**Merge Sort**

*   Average-case time complexity: O(n log n)

**Quicksort**

*   Best-case time complexity: O(n log n) (when the pivot be always the smallest or largest element)
*   Average-case time complexity: O(n log n)
*   Worst-case time complexity: O(n^2) (when the pivot be always the middle element)

Quicksort is like finding a good spot to divide the treasure chest, sorting each half, and then reassembling it.

**Which one be faster?**

In most cases, merge sort be faster than quicksort. But if ye have nearly sorted data or need to handle very large datasets efficiently, ye might want to consider another algorithm like heap sort.

To avoid quicksort's worst-case scenario:

*   Choose a good pivot strategy
*   Use an iterative implementation
*   Consider using hybrid sorting algorithms

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left_half = merge_sort(arr[:mid])
    right_half = merge_sort(arr[mid:])
    return merge(left_half, right_half)


def merge(left, right):
    merged = []
    while len(left) > 0 and len(right) > 0:
        if left[0] <= right[0]:
            merged.append(left.pop(0))
        else:
            merged.append(right.pop(0))
    merged.extend(left)
    merged.extend(right)
    return merged
